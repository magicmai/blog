# 《程序语言的奥妙：算法解读》
> #读书笔记 #算法 #算法入门

## 第1章 什么是算法
- **算法是“解决问题的处理步骤”**
- 烹饪用的食谱也是算法

- 算法是古老智慧的结晶，是程序的范本
- 学习算法才能编写出高质量的程序

- 算法的学习类似学习游戏攻略
- 算法是编写好程序的“棋谱”（步骤）

- 算法中 **“有效性”** 和 **“终止性”** 是必要的
- 断点是表示算法“有效性”的方法之一：断点被设置在算法的任意位置上，能够判断此位置是否满足给出的条件，即程序是否正确运行

- 重要的算法有 **技术计算、排序、查找、字符串模式匹配** 等。

- 专栏：算法的基础——结构化程序设计的思想：
    - 顺序结构：按照所述顺序处理。
    - 选择结构：根据判断条件改变执行流程。
    - 循环结构：当条件成立时，反复执行给定的处理操作。
  
## 第2章 变量和数组
- “数据”是各种信息的表现形式
- 算法表现为“处理”和“数据”的结合

- 处理信息的种类分组称为“数据类型”
- 常用的数据类型：“整数型”、“浮点型”、“字符型”、“字符串型”、“布尔型”

- 具体表示数据的定西就是“值”
- 字符型和字符串类型的值要用“'”或“"”符号括起来

- 变量是彷如数据（值）的箱子
- 变量中彷如数据（值）的操作称为代入

- 每个变量都有其相应的“变量名”
- 变量名一定要是唯一的

- 变量的值的代入是通过变量指定的赋值语句完成的
- 赋值语句的右边也可以是使用运算符号的公式

- 复制语句右边的变量名代表存储在此变量中的值
- 赋值语句右边也可以是包含变量的计算公式
 
- 变量也有数据类型
- 与变量的数据类型不一致的值不可以代入变量中

- 为了保存大量数据我们可以使用“数组”
- 被数组管理的数据必须是同种类型的值

- 每一个数组都有一个名字，这称为“数组名”
- 数组名必须唯一

- 构成数组的“箱子”的个数称为数组元素数
- 数组元素用元素编号（从头元素开始的序列号）进行管理

- 数组是存储大量相关联数据的储物柜
- 元素编号可以标识指定的数组元素

- 二维数组变量无缝隙地排列成长方形的形状
- 二维数组、三维数组等成为多维数组

- 二维数组利用“行”和“列”的元素编号相交叉的位置指定数组元素

- 在计算机世界中，字符串表现为“字符的连续排列”
- 字符串是字符型数组中的元素中存储字符的状态

- 字符串的长度由保存“字符长度”的变量管理
- 也可以在字符串的末尾加上“哨兵”进行管理：在字符串末尾的下一个位置存储一个字符编码，但此字符编码绝对不会出现在这个字符串中。一般情况下，取“哨兵”的值为数值0

- 专栏：常用的变量名：
    - 保存循环次数的变量名称：i，j，k
    - 使用数组下标的变量名称：index，idx
    - 数数时使用的变量名称：counter，count，cnt
    - 字符串处理中的变量的名称：str，string
 
# 第3章 数据结构
- 数据结构是为了实现大量数据的有效管理的机制，根据邮政编码的住址管理以及根据座位编号的学生管理都是数据结构
 
- 描述计算机算法的时候使用的数据结构有很多种
- 主要的数据结构有“数组”、“链表”、“栈”、“队列”和“树”
 
- 数据输入操作叫做 push，输出操作叫做 pop
- 栈的数据管理方法有 LIFO 和 FILO 两种
 
- 队列中先输入的数据先输出
- 队列的数据管理方法有 FIFO 和 LILO
 
- “链表”利用指针连接不连续的数据并管理这些数据的顺序，当“没有指向下一个数据的指针”时表示到达链表的尾部
 
- 单向链表的一个元素由数据和指向下一结点的指针构成
- 单向链表中要指出元素开头的头结点的指针（头指针）
 
- 双向链表中要指出指向头结点和尾结点的指针，且双向链表中的每个元素由数据和指向前后结点的指针构成
 
- 查找第N个元素时，利用数组中的下标可以直接查到，但在链表结构中查找时需要从第1个元素开始顺序遍历而花费不少时间，因此，这种情况下，数组比链表更有效率
 
- 执行数据的插入、删除操作时，利用链表的指针效率很高。但发生数据移动时数组花费的时间较多
 
- 环形缓冲是数组的头尾相连的数据结构，尾元素的下一个元素即为头元素
 
- 根据一个父结点和两个子结点的关系管理数据的结构称为二叉树
- 没有父结点的结点称为“根结点”，没有子结点的结点称为“叶结点”
 
- 适用于求最小值的数据结构——堆，是指“父结点的值不得比子结点的值大的二叉树”
 
- 元素为 N 的根数组，并且根数组的元素作为链表的头指针的结构为哈希表
- 将输入值通过哈希函数得到的值作为根数组中的元素编号
 
- 图是利用“结点”和“边”表示数据连接关系的数据结构
 
- 专栏：从 0 开始还是从 1 开始？
    - 在计算机开发初期把数组头元素考虑为 1 的编程语言：Fortran，Pascal，Basic（初期）
    - 后来的主流编程语言把数组头元素考虑为 0 ：C，C++，Java，VisualBasic（BASIC）
     
# 第4章 基本的算法
- 36、执行循环操作求 1~N 的和：变量 SUM 初始化为0，将 1~N 按顺序相加求其值

- 37、使用根数组保存数列的值： 斐波那契数列是开头元素是 0，第二个元素是 1，第三个元素以后的数列都是前两个元素相加的和按顺序排列起来的数列

- 38、用变量求数组元素值的和：求数组元素的和时，使用变量 SUM 并初始化为 0；在执行循环的过程中按顺序将元素的值求和

- 39、用计数器求数组的有效元素数：求数据个数时，使用变量 COUNT 并初始化为 0；未达到哨兵元素的时候，变量 COUNT 值加1（哨兵值时数列元素没有的值，它在“数组的末尾”，可以管理末尾数据。）

- 40、根据数组的各元素和以及元素个数求数组的平均值：在查找“哨兵”的过程中，计算 COUNT （元素个数）值和 SUM 值（和值），最后用SUM / COUNT 求平均值

- 41、求数组元素的最大值要使用保存最大值的变量：当变量 MAX 值初始化为比所有处理数据都小的值；当“数据元素 > MAX 值”时，令 MAX 值等于数组元素的值

- 42、使用变量保存最小值求数组的最小值：将变量 MIN 值初始化为比所有处理数据都打大的值；当“数组元素 < MIN 值”时，令 MIN 值等于数组元素的值

- 43、使用其他数组存放数组元素的顺序：利用初始化为 1 的数组，将对象数据数组的所有元素与另一个数组元素值相比较，当小的时候加 1，最后求出数组的元素排名

- 44、以秒为单位的时间大小的比较：虽然有按照时→分→秒的顺序比较大小的方法，但这样有点复杂；比较时分秒时，换算为“秒”再进行比较

- 45、求时间差时，将差再表示成时间形式：求时刻的差值应换算成以秒为单位的值；求以秒为单位的差值，再换算成“时分秒”格式

- 46、利用临时变量交换两个变量的值：为了交换两个变量的值，要使用临时变量；把先被覆盖的值提前存入临时变量中

- 47、用欧几里得相除法求2个数的最大公约数：欧几里得相除法定理：已知整数 X 和 Y （X >= Y），X 除以 Y 的余数为 R，X 和 Y 的最大公约数与 Y 和 R 的最大公约数相等。X 和 0 的最大公约数是 X。

- 专栏：代码和数据存在哪里？
    - 内存空间，“内存”的管理单位是“块”，所有的块都有对应的“物理地址”。如果把这些物理地址作为元素编号，就也可以把内存看成大规模的数组。

# 第5章 排序和搜索
- 排序是按照于对象数据所拥有的属性进行排序
- 排序顺序有“升序”（从小到大的顺序）和“降序”（从大到小的顺序）

**几种排序算法：桶排序、基数排序、简单选择法、简单交换法（冒泡排序）、简单插入法、希尔排序、归并排序、快速排序、堆排序等。**

- 准备空桶数组，在与数据对应号的桶中存放数据，把这些数据从桶的最先开始取出就是桶排序

- 根据各进制位的数值进行排序的是基数排序；K 位的数进行排序时需要执行 K 次的桶排序

- 简单选择法从“未排序部分”中找出最小值（或者最大值）并且移动到“已排序部分的末尾”

- 比较相邻的两个数，使两个数按照正确大小关系交换的方法就是简单交换法（冒泡排序）

- 简单插入法是向已排序部分的数列中的大小关系正确的位置插入数据的排序算法

- 希尔排序是把每隔一定距离的数据分为一组，组内通过比较大小进行排序

- 按升序排好序的多个数列中，最小值恒存在与各个数列的首位元素中

- 归并排序是利用归并算法进行排序；归并排序有二分和合并两个步骤

- 快速排序是很快的排序算法，重复根据基准值分为比基准值小的组和比基准值大的组进行排序

- 堆排序是利用堆的特性进行排序的算法**（？）**

**几种查找算法：线性查找、二分查找、简单的字符串查找、通过 KMP 查找、通过 BM 查找等。**

- 线性查找是从随机排列的数列中找出目标数据的算法

- 搜索对象数列在排好序的时候，可以利用二分查找快速地找出目标数据

- 在字符串查找算法中，需要考虑“由多个字符组成的字符串作为查找对象”

- 排在 KMP 法中决定了搜索局部字符串不一致的位置和从局部字符串的信息得到的接下来开始比较字符的位置

- BM 法有着从搜索的局部字符串的末尾到开头进行比较的特征

# 第6章 其他算法
- 牛顿法是通过“导函数”和“x，y 的变化量”表示斜切的斜率来求解高次方程的算法

- 高斯消元法是通过“前进消去”和“后退代入”两步来求解联合方程组的解

- 梯形法是通过计算梯形面积和来求得函数 f(x) 的定积分_____ 的近似值

- 用图来表示起始点、终点以及途中经过的节点和把它们连接在一起的路径，通过迪杰斯特拉算法求出最优路径

- “埃拉托色尼筛法”是从自然数中筛选掉不是素数的数来求素数的算法

- 递归调用类似于一条蛇吞下自己的尾巴，函数调用自身函数

# 第7章 算法的复杂度
- “时间复杂度”通过操作次数来测量
- “空间复杂度”通过使用内存的大小来测量

- 复杂度与数据量成比例的时候，即处理的数据量很大的时候，此算法是效率非常低的算法

- 算法的复杂度是通过处理对象的数据量 N 为基准的 _O_ 记法来表示的
